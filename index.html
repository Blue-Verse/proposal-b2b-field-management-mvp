핵심 요구사항 분석

  • 관리자(소장)용 대시보드 — 현장·근로자 정보(DB) 관리, 현장별 고유 단축 URL 자동 생성, 알림톡/SMS 발송 연동, 실시간 응답 데이터 수집·현황 확인, 엑셀 다운로드
  • 작업자 수신용 모바일 웹 폼 — 앱 설치/로그인 없이 URL 클릭만으로 즉시 폼 진입, 텍스트·체크박스 입력, 스마트폰 현장 사진 첨부·제출
  • FE/BE 완전 분리 아키텍처 — 향후 타 B2B SaaS와의 연동 및 확장을 고려한 RESTful API 기반 설계
  • 반응형 웹 — PC Web(Chrome, Edge) + 모바일 Web(iOS, Android 기본 브라우저) 최적화
  • 풀 라이프사이클 — 상세 기획 + UI/UX 디자인 + FE/BE 개발 + 서버/DB/인프라 구성

제안 포인트

본 프로젝트는 관리자 대시보드 + 현장 모바일 폼이라는 전형적인 B2B 현장 관리 패턴입니다. 유사 구조의 프로젝트를 다수 수행한 경험을 바탕으로 다음과 같이 제안합니다:

  • 토큰 기반 접근 구조 — EZ-Approve 프로젝트에서 구현한 토큰 기반 외부 포탈 패턴을 적용하여, 작업자가 로그인 없이 고유 URL로 즉시 폼에 진입하는 구조를 효율적으로 구축
  • 알림톡/SMS 연동 경험 — 뿌리오(Ppurio) API 연동 경험을 활용하여 현장별 단축 URL 발송 기능을 안정적으로 구현
  • B2B SaaS 확장 설계 — Harmony Link의 멀티테넌트 아키텍처 경험을 기반으로, 향후 다수 고객사 운영을 고려한 확장 가능한 데이터 모델 설계
  • 기술 내재화 지원 — 내부 주니어 FE 개발자와의 코드 리뷰 및 문서화를 통해 프로젝트 완료 후 자체 유지보수가 가능한 체계 구축

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

시스템 소프트웨어 개발 전문 법인에서 신규 B2B 사업의 일환으로 '웹 기반 현장 관리/수발신 MVP'를 구축합니다. 핵심 목표는 현장 작업자가 별도의 앱 설치 없이 URL 접속만으로 간편하게 업무를 처리하는 것입니다.

  • 기존 문제: 현장 작업자에게 앱 설치를 요구하면 채택률이 낮아짐
  • 해결 방안: URL 기반 모바일 웹 폼으로 진입 장벽 제거
  • 향후 비전: MVP 검증 후 B2B SaaS로 확장, 다수 고객사 운영

2. 사용자 정의

사용자 유형설명주요 기능
관리자(소장)현장을 관리하고 작업자에게 업무를 배포하는 담당자현장/근로자 관리, 단축 URL 생성, 알림톡/SMS 발송, 응답 데이터 조회, 엑셀 다운로드
현장 작업자현장에서 업무를 수행하고 결과를 보고하는 실무자URL 접속 → 폼 입력 → 사진 첨부 → 제출 (로그인 불필요)
시스템 관리자전체 시스템 운영 및 설정 담당관리자 계정 관리, 시스템 설정, 알림 서비스 설정

3. 기능 요구사항

3.1 관리자 대시보드

  • [필수] 현장 정보 CRUD (현장명, 주소, 담당자 등)
  • [필수] 근로자 정보 CRUD (이름, 연락처, 배정 현장 등)
  • [필수] 현장별 고유 단축 URL 자동 생성 (+ QR코드)
  • [필수] 단축 URL을 알림톡/SMS로 대량 발송
  • [필수] 작업자 응답 데이터 실시간 수집 및 현황 대시보드
  • [필수] 수집 데이터 엑셀(Excel) 파일 다운로드
  • [선택] 응답 데이터 필터/검색/정렬

3.2 작업자 모바일 웹 폼

  • [필수] URL 접속 시 즉시 폼 표시 (로그인/설치 없음)
  • [필수] 텍스트 입력, 체크박스 선택 등 동적 폼 렌더링
  • [필수] 스마트폰 카메라로 현장 사진 촬영 및 첨부
  • [필수] 폼 제출 시 서버로 데이터 실시간 전송
  • [선택] 오프라인 임시 저장 (PWA Service Worker)

3.3 알림/메시징

  • [필수] 알림톡/SMS API 연동 (뿌리오 또는 NHN Cloud)
  • [필수] 발송 이력 관리 및 실패 재발송
  • [선택] 발송 예약 기능

4. 비기능 요구사항

항목요구사항
성능API 응답 시간 500ms 이내, 모바일 폼 로딩 2초 이내
보안HTTPS 필수, JWT 인증, SQL Injection/XSS 방어, 파일 업로드 검증
확장성RESTful API 분리로 향후 모바일 앱/타 SaaS 연동 가능
호환성Chrome, Edge 최신, iOS/Android 기본 브라우저 반응형 지원
접근성모바일 터치 영역 최소 44px, 폼 라벨 명확, 색상 대비 WCAG AA

5. 기술 제약사항

  • 프런트엔드: React (클라이언트 요구사항 반영)
  • 백엔드: Node.js (NestJS) — 클라이언트 내부 FE 개발자 기술 내재화 고려
  • FE/BE 완전 분리 RESTful API 아키텍처 (필수)
  • 향후 B2B SaaS 확장을 위한 멀티테넌트 고려 설계

6. 범위 외 사항 (Out of Scope)

  • 네이티브 모바일 앱 개발 (iOS/Android)
  • 결제/과금 시스템
  • 실시간 채팅/메신저 기능
  • GPS 기반 위치 추적
  • 다국어 지원
