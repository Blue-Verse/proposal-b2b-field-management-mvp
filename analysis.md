### 클라이언트 질문 답변

**[질문 1] (아키텍처 검증)**

프론트엔드는 React(Next.js), 백엔드는 Node.js(NestJS)를 주력으로 사용하며, FE/BE를 완전히 분리한 RESTful API 아키텍처를 다수 구축한 경험이 있습니다. 대표적으로 전자결재 SaaS 프로젝트(Next.js 13 + NestJS 10, 120-150개 API 엔드포인트, 50+ 페이지)와 VC 펀드 관리 플랫폼(200-300+ API 엔드포인트, 22개 도메인 모듈)을 100% 전담하여 설계·개발·배포하였습니다. 두 프로젝트 모두 OpenAPI 기반 API 명세서를 작성하고, JWT 인증 + RBAC 권한 관리를 적용한 분리 아키텍처입니다. 본 프로젝트에서도 동일한 구조로 React + NestJS RESTful API를 구축하며, Swagger/OpenAPI 문서를 자동 생성하여 향후 B2B SaaS 연동에 즉시 활용 가능한 구조를 제공하겠습니다.

**[질문 2] (UX 및 모바일 최적화 검증)**

유사 경험이 있습니다. 전자결재 SaaS 프로젝트에서 토큰 기반 외부 포탈을 구현하여, 외부 사용자가 별도 로그인이나 앱 설치 없이 고유 URL만으로 즉시 폼에 진입하여 데이터를 제출하는 구조를 구축하였습니다. 또한 시니어 케어 관리 플랫폼에서 모바일 환경에서의 사진 촬영·첨부·제출 기능과 실시간 데이터 수집을 구현한 경험이 있습니다. 두 프로젝트 모두 모바일 환경 최적화(터치 영역, 반응형 레이아웃, 이미지 압축 업로드)를 적용하였으며, 폼 데이터는 REST API를 통해 즉시 서버로 전송되어 관리자 대시보드에서 실시간 확인이 가능합니다. PWA 적용 경험도 보유하고 있어, 오프라인 임시 저장 등 현장 환경에 맞는 최적화를 제안드릴 수 있습니다.

**[질문 3] (기술 내재화 및 인수인계 검증)**

기술 내재화를 위해 다음과 같은 체계적인 인수인계 프로세스를 제안합니다. 첫째, 개발 기간 중 내부 주니어 FE 개발자분과 주 2회 GitHub PR 코드 리뷰 세션을 진행하여, 코드 구조와 설계 의도를 실시간으로 공유합니다. 둘째, 프로젝트 완료 시 API 명세서(Swagger), DB 스키마 문서, 배포 가이드, 아키텍처 다이어그램 등 개발 문서를 체계적으로 정리하여 전달합니다. 셋째, 배포 단계에서 2회의 기술 설명 세션(프론트엔드 구조 + 백엔드 API 구조)을 진행하여, 내부 팀이 독립적으로 유지보수할 수 있는 수준까지 지원합니다. 이전 프로젝트에서도 동일한 방식으로 인수인계를 완료한 경험이 있으며, 1개월 무상 하자 보수 기간 내 추가 문의에도 적극 대응하겠습니다.
